# Mollie Terminal for WooCommerce 0.2.0

This release automates the in-person payment flow and makes terminal selection self-service, so cashiers no longer babysit the payment buttons.

## Added

- **Automatic payment flow.** After "Start Terminal Payment," the checkout polls Mollie on its own — showing "Sending to terminal…" then "Waiting for terminal…" — and completes the order automatically once the terminal confirms.
- **Live terminal dropdown.** The terminals connected to your Mollie account are fetched from the Mollie Terminals API and shown as a dropdown at checkout, with the configured default preselected. The settings screen offers the same fetched dropdown for choosing the default terminal.
- **Test-mode warning.** The settings screen now warns that Mollie terminals are only available on live accounts.

## Removed

- The Mollie **Profile ID** setting. It is not required for point-of-sale payments, and terminals are now listed across the whole account.

## Notes

- The webhook URL is sent with every payment, so no Mollie dashboard webhook configuration is required.
- A test API key cannot drive a physical terminal — use Live mode with your live API key.
