# Mollie Terminal for WooCommerce 0.1.0

This is the first public release of Mollie Terminal for WooCommerce.

## Highlights

- Adds a Mollie Terminal payment gateway for WooCommerce and WooCommerce POS.
- Starts Mollie `pointofsale` payments on a configured Mollie terminal.
- Treats Mollie as the source of truth for payment and refund state.
- Uses per-order locks and payment attempt history to reduce duplicate payment and refund risks.
- Reconciles webhook, polling, cancellation, retry, and refund paths by fetching current Mollie state before changing WooCommerce orders.

## Included in this release

### Payment collection

- Create Mollie point-of-sale payments for WooCommerce orders.
- Validate configured terminal IDs before dispatching a payment.
- Store Mollie payment IDs and append-only payment attempt history on the order.
- Reuse active open/pending payment attempts instead of creating duplicate terminal payments.
- Allow retry after final unpaid states such as failed, canceled, or expired.

### Webhooks and polling

- Adds a Mollie webhook handler for payment updates.
- Uses the webhook payment ID to fetch authoritative payment details from Mollie.
- Verifies amount, currency, method, mode, and order association before completing an order.
- Adds POS polling so the cashier interface can show current payment status while webhooks remain authoritative.

### Cancellation and refunds

- Supports canceling current terminal payments when Mollie reports they are cancelable.
- Fetches and reconciles payment state when cancellation races with terminal completion.
- Supports WooCommerce refunds through Mollie refunds.
- Prevents duplicate refunds by storing WooCommerce refund attempt metadata and reconciling existing Mollie refunds before creating a new one.
- Rejects over-refunds using Mollie refund state rather than local WooCommerce metadata alone.

### Admin and diagnostics

- Adds WooCommerce payment settings for Mollie mode, API key, profile ID, default terminal ID, and webhook URL.
- Shows basic diagnostics for active environment, profile ID, selected terminal, webhook URL, pairing availability, last API error, and last webhook event.
- Redacts sensitive API key material from logs.

### Safety model

- Local WooCommerce metadata is treated as a cache.
- Mollie payment/refund state fetched from the API is authoritative.
- Per-order locks protect create-payment, cancel-payment, and refund operations.
- Payment attempts are preserved as append-only history for auditability.
- POS payment amounts are constrained to EUR, matching Mollie's documented point-of-sale currency support.

## Requirements

- WordPress with WooCommerce installed and active.
- PHP 7.4 or newer.
- A Mollie account with point-of-sale access.
- A Mollie profile with Point of Sale enabled.
- A Mollie terminal ID for the relevant test or live profile.

## Known operational notes

- Test-mode terminals are created by enabling Point of Sale on a Mollie test profile and retrieving the test terminal via Mollie's terminal listing flow.
- Mollie's terminal pairing-code endpoints do not currently support test mode, so live pairing-code behavior and test terminal setup are intentionally different.
- Android Tap to Pay app-to-app intent integration is separate from this WordPress/WooCommerce terminal gateway.
